THE METHOD

How a question becomes a grounded, cited answer — the retrieval & generation pipeline.
1

Query expansion deep

HyDE writes a hypothetical abstract for your question, plus multi-query paraphrases — so retrieval matches academic phrasing, not just your wording.

2

Hybrid retrieval

BGE-M3 produces dense (semantic) and sparse (lexical) vectors; both are searched in Qdrant, so exact terms and meaning both count.

3

Reciprocal Rank Fusion

Merges the dense, sparse, and per-variant rankings into one list — no score-scale tuning required.

4

Cross-encoder rerank

A Jina reranker jointly scores each (query, passage) pair for precise relevance — far sharper than cosine similarity.

5

Citation-graph rerank

Re-orders by α·relevance + β·log(citations) + γ·recency, surfacing influential and current work — not just topically similar text.

6

Quality gate CRAG

The retrieved context is scored for sufficiency; if it's too weak the system abstains ("insufficient evidence") instead of guessing.

7

Grounded generation

A Groq LLM (Llama-3.1-8B fast / 3.3-70B deep) writes a literature review with an inline [arXiv id] citation for every claim.

8

Claim verification

Each sentence is entailment-checked against the sources; unsupported claims are dropped — lifting measured faithfulness +56%.

Runs local-first (BGE-M3 + Ollama) or scales to hosted APIs (Groq · Jina · Qdrant Cloud) by config. Deep dive → ARCHITECTURE.md
